# 2024/11/12 agenda

## 何作りたいの
既存アプリである[ルーチンタイマー](https://play.google.com/store/apps/details?id=com.freecolors.routinetimer&hl=ja)のクローンアプリ（Android）。
### 言葉の定義
「タスク」は、以下の要素を子に持つ
- 名前
- 所要時間

「ルーチン」は、以下の要素を子に持つ
- 名前
- タスクの単方向リスト

### 要求する機能要件
- ルーチンの作成/編集とローカルDB保存
- 保存したルーチンの閲覧
- 保存したルーチンの実行
    - タスクの所要時間ぶんのタイマーが自動で走る
    - タスクの所要時間が経過すると、次のタスクのタイマー開始
    - タスク開始時、タスクの名前を音声読み上げ（TextToSpeech）
    - タイマーは一時停止可能
    - タイマーはスキップ可能
        - スキップすると次のタスクのタイマー開始

## なぜ作りたいの
主な動機は二つ
- 普段使いしている「[ルーチンタイマー](https://play.google.com/store/apps/details?id=com.freecolors.routinetimer&hl=ja)」の出来に満足していないから（自分で機能追加する土台としてまずクローンを作りたい）
- いちばん普段使いしているアプリが自家製だったら超かっこいいから

上記がプリミティブな動機で、そこから派生して
- 「最近Android開発触れてなかったし触りたい」
- 「いい加減そろそろ人間の普段使いに耐えうるアプリをポートフォリオに載せたい」

などもあります。


## 何が不安なの
- プロジェクトの進め方
    - 何から着手？
    - 調査が必要そうなものは何？
    - 所要時間の見積もり、ペースメイクなど
- 設計
- 詰まった時に詰まりっぱなしになって萎えること
- 自分の決断に自信が持てない時にウダウダ言って進まなくなること

## 協力者に何を期待するの
- 詰まった時の相談（非同期も同期も両方お願いしたくなりそうです）（同期的な相談は週1程度できたら最高です）
    - 同期的な相談は１回あたりの時間見積もれるようにしておこう
- 「私だったらこうする」の知見シェア
    - テキストなどで「これからこうしようと思ってますがどう思いますか」を持っていくので見てほしいです
- たまにペアプロなど
    - 基本ドライバーずっと私で、変なことしてないか見て欲しいです
- そのほか、開発から普段使いに至るまでのフロー中に困った時の相談など
## ちなみに既存アプリの何に納得いってないの
「最初は追加機能なしの最低限クローンをまず作りたい」というのを前提に、その完成後に組み込みたい既存アプリの不満点を以下に挙げます

- ルーチン入力中に、手動で「保存」を押すまでデータベースに保存されない
- ルーチンを入れ子構造にしたい（新しいルーチンの入力時に、既に完成した一連のルーチンを呼び出したい）
- ルーチン実行中の「停止」「スキップ」について統計をとりたい
